# Public-data Update Preflight

Use this template to run the ACT-11 public-data update preflight
before any manual `git add` / `git commit` / `git push` that touches
`public-data/` or `site/index.embedded.html`.

**Audience:** local-hermes, authorized primary agent, or the human
orchestrator. Do **not** send this to a trial agent.

**What it does:**

1. Reads `config/public-data-export-plan.yml` for the export scope.
2. Snapshots the current `public-data/` state.
3. Regenerates `public-data/` from `data/` via
   `scripts/export_public_data.py --plan ... --replace`.
4. Validates `public-data/` and rebuilds `generated/index.json` and
   `site/index.embedded.html`.
5. Runs regression checks (1-project downgrade, BookTrans Desk repo
   misclassification, redaction FAIL > 0).
6. Writes a reviewable artifact directory at
   `artifacts/public-data-update-preflight/`.

**What it does NOT do:**

- `git add` anything
- `git commit`
- `git push`
- Touch Cloudflare Pages
- Expose `data/` or `generated/`

## Pre-flight (human)

Before running this preflight, make sure:

- [ ] You are on the `main` branch.
- [ ] Your local `main` is up to date with `origin/main`
      (`git pull --ff-only`).
- [ ] `data/` is the version you intend to publish.
- [ ] `config/public-data-export-plan.yml` still reflects the intended
      project/agent scope.

## Command

```bash
cd ~/workspace/projects/agent-project-control-tower
make public-update-preflight
```

or equivalently:

```bash
python3 scripts/public_data_update_preflight.py \
  --plan config/public-data-export-plan.yml \
  --output artifacts/public-data-update-preflight
```

## What to do with the result

1. Open `artifacts/public-data-update-preflight/UPDATE_SUMMARY.md`.
   If it says **FAIL**, do **not** commit. The summary tells you
   which check failed.
2. Open `PUBLIC_DATA_DIFF.md` and confirm the project/agent/event
   counts match your intent.
3. Open `REVIEW_CHECKLIST.md` and walk through it.
4. If BookTrans Desk is in the plan, confirm:
   - `repo` = `conanxin/booktrans-desk` (not conanxin-homepage)
   - current phase = `S13` (not HP-33)
   - `source_commit` = `16f38b6`
5. Run `git diff public-data/` to see the actual file changes.
6. When satisfied, run the **commit-and-push** sequence from
   `NEXT_STEPS.md` (in the same artifact directory).

## Never

- `git add .` — stage only what you reviewed
- `git add data/` — `data/` is gitignored and must stay private
- `git add generated/` — build artifact, regenerated by CI
- `git add artifacts/` — review-only, never committed
- `git push --force`
- Push to `master` (we don't have a master branch; the primary branch
  is `main`)
