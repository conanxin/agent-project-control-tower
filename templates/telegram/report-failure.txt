Report a phase failure in the Agent Project Control Tower.

Use this when a phase did NOT achieve its goal. Silent blockers are
the most expensive thing in a multi-agent system. Report them.

Run on the agent's own machine, inside the tower repo:

  cd ~/workspace/projects/agent-project-control-tower

Fill in the placeholders before sending.

  PROJECT_ID      = <PROJECT_ID>
  AGENT_ID        = <AGENT_ID>
  PHASE_ID        = <PHASE_ID>
  PHASE_NAME      = "<short name>"
  SUMMARY         = "<1-2 sentences, no secrets/paths/IPs>"
  FAILURE_REASON  = "<root cause, no secrets/paths/IPs>"
  NEXT            = "<what's next, no secrets/paths/IPs>"
  SOURCE_REPO     = <owner>/<repo>
  SOURCE_COMMIT   = <real commit hash from git log of SOURCE_REPO>

Command:

  python scripts/tower.py report-failure \
    --project-id       <PROJECT_ID> \
    --agent-id         <AGENT_ID> \
    --phase-id         <PHASE_ID> \
    --phase-name       "<short name>" \
    --summary          "<1-2 sentences>" \
    --failure-reason   "<root cause>" \
    --next             "<what's next>" \
    --source-repo      "<owner>/<repo>" \
    --source-commit    <SOURCE_COMMIT>

This produces a PHASE_REPORT event with:

  status = FAIL
  health = red

Hard rules:

  - Do NOT paste real tokens, real home paths (/home/<user>/...),
    real IPs, or .env references in --failure-reason. Refer to the
    failure by phase id and a short tag.
  - Do NOT run git add . in this repo. Always list files explicitly.
  - data/ is gitignored. Do not attempt to commit it.
