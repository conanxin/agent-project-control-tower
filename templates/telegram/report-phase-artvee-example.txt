Report a phase completion in the Agent Project Control Tower.

This template is the Artvee Gallery P5F worked example referenced from
https://control-tower.conanxin.com/help/  (section: 完整示例 → Step 2)
and docs/AGENT_USAGE_PLAYBOOK.md §17.

All values below are PLACEHOLDERS. Replace them before sending, or
generate a single-line command via:

  python scripts/generate_tower_command.py report-phase \
      --project-id     <PROJECT_ID> \
      --agent-id       <AGENT_ID> \
      --phase-id       <PHASE_ID> \
      --phase-name     "<short name>" \
      --status         <PASS|PARTIAL|FAIL|BLOCKED|PAUSED|SKIPPED> \
      --health         <green|amber|red|gray> \
      --summary        "<1-2 sentences, NO secrets/paths/IPs>" \
      --next           "<what comes next>" \
      --next           "<another next-step if needed>" \
      --source-repo    <owner>/<repo> \
      --source-commit  <commit hash from git log of SOURCE_REPO>

DO NOT replace <PLACEHOLDER> tokens in THIS template by hand — the
generator exists so that the tower.py CLI and the template do not drift.

# Hard rules (DO NOT violate)

  * DO NOT add --source-commit-url unless you have the real commit URL.
  * DO NOT include /home/..., /Users/..., IPv4, .env, sk-..., ghp_...,
    Bearer tokens, or any other secret in --summary / --next / --phase-name.
  * DO NOT run export_public_data.py, git add public-data/, or git push
    on the tower repo. That is the SECOND door — human / authorized
    primary agent only. See /help/ §6 (双门模型) for details.
  * ACT-13D: prefer Chinese for --summary, --next, --phase-name. The
    dashboard will display the Chinese text first; the original English
    (if any) hides behind a "原文" toggle. --project-id / --agent-id /
    --source-repo / --source-commit stay verbatim. If you send English
    today, the dashboard still works — but a follow-up commit to
    apps/dashboard/src/lib/localized-content.ts will be needed to map
    the new (project_id, phase_id) pair into Chinese. See /help/ §10
    ("为什么有些字段仍保留英文？").
  * After the event lands in data/, report back the exact event JSON
    path:
      data/events/<TIMESTAMP>__PHASE__<AGENT_ID>__<PROJECT>__<PHASE>.json
    The human reviewer will open it before approving the export.
